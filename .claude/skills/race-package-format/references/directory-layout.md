# Directory Layout вҖ” Canonical Trees

## Minimal race (the smallest valid package)

```
hello-world/
в”ңв”Җв”Җ package.json
в”ңв”Җв”Җ race.ts
в”ңв”Җв”Җ prompts/
в”Ӯ   в””в”Җв”Җ 01_greet.md
в”ңв”Җв”Җ tsconfig.json
в””в”Җв”Җ README.md
```

A minimal race has one prompt runner writing one artifact. No schemas, no templates.

```
codebase-discovery/

в”ңв”Җв”Җ prompts/
в”Ӯ   в”ңв”Җв”Җ 01_inventory.md
в”Ӯ   в”ңв”Җв”Җ 02_entities.md
в”Ӯ   в”ңв”Җв”Җ 03_services.md
в”Ӯ   в””в”Җв”Җ 04_report.md
в”ңв”Җв”Җ schemas/
в”Ӯ   в”ңв”Җв”Җ inventory.ts
в”Ӯ   в””в”Җв”Җ entities.ts
в”ңв”Җв”Җ templates/
в”Ӯ   в””в”Җв”Җ report.html.ejs
в”ңв”Җв”Җ examples/
в”Ӯ   в””в”Җв”Җ sample-output.html
в”ңв”Җв”Җ README.md
в”ңв”Җв”Җ tsconfig.json
в””в”Җв”Җ dist/                  # generated by `tsc` at publish time, .gitignored
    в”ңв”Җв”Җ race.js
    в””в”Җв”Җ race.d.ts
```

Use `schemas/<name>.ts` for any Zod schema referenced by more than one step. Inline schemas in `flow.ts` only when the shape is small (вүӨ5 fields).

`templates/` is optional вҖ” used when a step generates a structured artifact (HTML report, markdown doc) and the prompt fills a template. The template can be EJS, Mustache, or just a plain file with `{{tokens}}` вҖ” the prompt knows how to fill it.

`examples/sample-output.<ext>` is one or more rendered artifacts the README can link to. Especially valuable for HTML reports вҖ” the catalog page can iframe them.

## Generator template (in `packages/generator/templates/<name>/`)

```
discovery/                # the template name
в”ңв”Җв”Җ package.json          # with {{tokens}} for pkgName etc.
в”ңв”Җв”Җ race.ts               # with {{tokens}} for runnerNames etc.
в”ңв”Җв”Җ prompts/
в”Ӯ   в”ңв”Җв”Җ 01_inventory.md
в”Ӯ   в”ңв”Җв”Җ 02_entities.md
в”Ӯ   в”ңв”Җв”Җ 03_services.md
в”Ӯ   в””в”Җв”Җ 04_report.md
в”ңв”Җв”Җ schemas/
в”Ӯ   в”ңв”Җв”Җ inventory.ts
в”Ӯ   в””в”Җв”Җ entities.ts
в”ңв”Җв”Җ tsconfig.json
в””в”Җв”Җ README.md             # with В§7.4 sections pre-filled with {{tokens}}
```

Templates use `{{token}}` placeholders that the scaffolding engine (`packages/generator/src/scaffold.ts`) replaces at generation time. Tokens supported in v1:

- `{{pkgName}}` вҖ” the package name (e.g., `@ganderbite/relay-foo`)
- `{{raceName}}` вҖ” the bare race name (kebab-case)
- `{{runnerNames}}` вҖ” an array of runner names; access via index `{{runnerNames[0]}}`
- `{{description}}` вҖ” one-line race description from the user prompt
- `{{audience}}` вҖ” `pm | dev | both`

Templates must compile and run AS-IS before substitution вҖ” i.e., the scaffolding engine produces a valid race without the user editing further. The placeholders are syntactically valid TS strings.

## What stays out of the published package

The `package.json` `files` array determines what gets published to npm. Keep it tight:

```json
"files": ["dist", "prompts", "schemas", "templates", "examples", "README.md"]
```

Notably excluded: `race.ts` source (only `dist/` is shipped), `tsconfig.json` (build artifact, not runtime).

`.gitignore` should cover:
```
node_modules/
dist/
.relay/
*.log
.DS_Store
```
