# hello-world

The smallest valid Relay flow. Use it to verify your local setup, to read a minimal end-to-end example, or as the starting point for a fork.

## What it does

Two prompt steps. The first step (`greet`) reads your `--name` input and produces a JSON handoff with a single friendly sentence. The second step (`summarize`) reads that handoff and writes a markdown file titled `greeting.md` into the run's `artifacts/` directory.

That is the whole flow. It exercises the three primitives a real Relay flow relies on: input variables, handoff routing between steps, and artifact output.

## Sample output

After a run, `artifacts/greeting.md` contains a short markdown document like:

```markdown
# Hello, Ada

> Welcome to Relay, Ada вЂ” glad to have you here.

This flow ran two prompt steps. The first produced a JSON handoff with a
greeting, and the second turned that handoff into this markdown artifact.
```

The exact wording varies per run because the greeting sentence is generated by the model.

## Install

This flow ships inside the Relay monorepo as a local example. There is no published package. To run it from a clone of the repo:

```bash
pnpm install
pnpm --filter hello-world build
```

The build step compiles `flow.ts` to `dist/flow.js`, which is the artifact the CLI loads.

## Run

From the repo root:

```bash
relay run ./examples/hello-world --name="Ada"
```

The path form (`./examples/hello-world`) tells the CLI to resolve the flow from a local directory instead of looking it up in the catalog. When the run completes, the generated markdown lands in `.relay/runs/<runId>/artifacts/greeting.md`.

## Configuration

The flow accepts one input:

| Field | Type | Default | Notes |
|---|---|---|---|
| `name` | `string` | (required) | The name of the person to greet. |

There are no per-step model overrides. Both steps run on whichever model your provider selects by default.

## Environment

This flow runs on your Claude subscription. Run `claude /login` once to authenticate before your first run.

Run `relay doctor` before your first run to confirm Node, the `claude` binary, auth state, and the `.relay` directory are all in order.

## Customization

Fork the flow by copying the directory:

```bash
cp -r examples/hello-world ./my-flow
cd ./my-flow
```

Then edit:

- `prompts/01_greet.md` вЂ” change what the first step writes into the handoff.
- `prompts/02_summarize.md` вЂ” change the markdown template the second step produces.
- `flow.ts` вЂ” add inputs to the Zod schema, add a third step, or swap the artifact extension.

Rebuild with `pnpm --filter my-flow build` after any change to `flow.ts`.
